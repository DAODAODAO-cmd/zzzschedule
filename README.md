# 🛒 谷子排单簿

把评论、调价、排单、推车图和收款统计装进同一本小簿子。

它不要求安装软件，也没有复杂后台。打开网页，粘贴评论，就能得到可以继续编辑的排单表；电脑上认真排，手机上随时补，两端还能通过 Supabase 同步。

## 它能做什么

- 批量解析评论，新的排单会继续累加，不覆盖旧数据
- 像表格一样直接改名字、拖动换位
- 入均、加减、固定价三种调价方式
- 自动计算均价、成配情况和每个人应付金额
- 一键导出完整排单图和固定版式的推车图
- 推车图支持红加、蓝减、橘色入均、群二维码和缺配提示
- 多表页切换，每一团的数据互不干扰
- 导入 Excel、导入备份、导出本表或全部数据
- 手机清爽版，只留下评论、排单和推车功能
- Supabase 云同步，图片压缩后单独存储，减少数据库占用

## 一分钟开排

1. 打开 `index.html`。
2. 在“角色 / 谷子列设置”中添加角色和图片。
3. 把评论粘贴到“批量粘贴评论”。
4. 检查预览后追加到排单表。
5. 直接修改表格，最后导出排单图或推车图。

数据默认保存在浏览器中。换设备或清理浏览器前，请先导出备份；也可以配置云同步。

## GitHub Pages 发布

仓库上传完成后，进入：

`Settings → Pages → Build and deployment → Deploy from a branch`

选择 `main` 分支和根目录，保存后即可得到公开网址。项目没有构建步骤，GitHub Pages 可以直接运行。

## Supabase 云同步

项目已填入对应的 Supabase Project URL 和 publishable key。首次启用还需要：

1. 打开 Supabase 项目的 SQL Editor。
2. 复制并执行 [`supabase_setup.sql`](./supabase_setup.sql)。
3. 回到网页，点击“新建同步码”。
4. 第一次点击“上传云端”。
5. 在另一台设备输入相同同步码，再点击“读取云端”或开启“自动同步”。

同步码相当于这份排单的钥匙，请不要公开。publishable key 本来就用于前端；真正不能放进仓库的是 `service_role` key，本项目没有使用它。

### 节省云端空间

- 角色图进入浏览器时会先缩小和压缩
- 云端数据表只保存排单数据与图片网址
- 图片使用内容摘要命名，相同图片不会反复生成不同文件
- 单张云端图片限制为 1 MB

## Excel 导入格式

第一行放角色名，每个角色占一列。可以额外加入“调价”和“单价”两行，后续行填写排单名字。一个 Excel 工作簿有多张工作表时，会分别导入成多个表页。

## 隐私提醒

排单可能包含群昵称、二维码和价格信息。公开演示或提交截图前，请先检查并遮挡私人内容。群二维码上传到公开图片空间，但文件路径包含不可预测的同步码和图片摘要；不要把同步码发到公开页面。

## 浏览器支持

推荐使用最新版 Chrome、Edge 或手机系统浏览器。截图识别、Excel 导入、图片导出和云同步依赖网络加载相应组件。
